# 功能计数技术实现总结

本文整理当前项目里所有主要“计数/次数/数量/索引/上限”的实现位置，方便后续继续改规则、调 UI、优化性能时快速定位。

## 1. 总览

| 类型 | 主要实现 | 当前含义 |
| --- | --- | --- |
| 玩家轮转 | `scripts/GameManager.gd` 的 `player_count`、`current_player` | 当前按 2 人对局循环，超过 `player_count` 后回到玩家 1。 |
| 本回合步数 | `GameManager._turn_move_count` | 统计当前玩家本回合已经执行的动作数，用于撤回、结束回合、连跳限制和冲刺跳上限。 |
| 本回合动作类型 | `GameManager._turn_move_kind` | 记录本回合首个动作是 `step` 还是 `jump`，防止走一步后继续跳，或跳后改走普通步。 |
| 悔棋栈 | `GameManager._turn_rollback_stack` | 每次本地行动前存一次快照，栈大小决定是否可以撤回本回合动作。 |
| 对局记录步数 | `GameManager._match_record.entries` | 每条 move/end_turn 记录都有递增 `index`，UI 显示“已记录 N 步”。 |
| 回放步数 | `GameManager._replay_step_index` | 表示当前回放到第几条记录，范围被夹在 `0..entries.size()`。 |
| 棋盘格数量 | `BoardManager.center_radius=4`、`arm_size=4` | 当前中心六边形 61 格，六个三角臂共 60 格，总计 121 格。 |
| 玩家棋子数量 | `BoardManager.player_starts` | 每个玩家起始臂为 10 格，所以默认每名玩家 10 枚棋子。 |
| 被动技能数量 | `BoardManager.assign_random_passive_skills()` | 每个玩家至少 3 枚棋子时，随机分配 3 个特殊棋子。 |
| 冲刺跳次数 | `PassiveSkillRuleEngine.DASH_JUMP_MAX_HOPS=2` | 冲刺跳棋子每回合最多参与 2 次跳跃动作。 |
| 可达格统计 | `PassiveSkillRuleEngine.get_turn_reachable_coords()` | 用字典去重后统计观察模式下“可达 N 格”。 |
| AI 搜索深度 | `AIPlayer.max_jump_depth=8` | AI 连跳路径递归最多搜索 8 层。 |
| 渲染帧率 | `project.godot`、`BackgroundManager.target_fps=60` | 项目与运行时都限制到 60 FPS。 |
| 场景实例数量 | `BackgroundManager` | 草叶、树、萤火、棋盘光点、石柱光等由固定数量或数组驱动。 |
| 音效播放器池 | `AudioManager._effect_players` | 创建 6 个音效播放器，使用 `_next_effect_player` 循环复用。 |
| 网络包轮询 | `NetworkManager` | 每帧循环读取 WebSocket 和 WebRTC 当前可用包数量。 |

## 2. 回合与玩家计数

### 玩家轮转

实现位置：`scripts/GameManager.gd`

核心变量：

```gdscript
@export var player_count: int = 2
var current_player := 1
```

本地模式 `_end_turn()` 与在线权威逻辑 `_online_advance_turn_locally()` 都使用同一套循环逻辑：

```gdscript
current_player += 1
if current_player > player_count:
	current_player = 1
```

当前技术含义：

- `player_count` 是玩家数量上限，目前默认 2。
- `current_player` 是当前行动玩家编号。
- 如果以后要支持 3/4/6 人局，除了改 `player_count`，还需要同步扩展 `BoardManager._build_player_regions()` 的起始区/目标区配置。

### 回合令牌

实现位置：`scripts/GameManager.gd`

变量：

```gdscript
var _turn_token := 0
```

用途：

- 每次重新开局、切换 AI 托管、开始新回合、进入回放等都会递增。
- AI 等待、AI 执行动作、回放播放这些异步流程会检查 token。
- 如果旧流程的 token 与当前 `_turn_token` 不一致，就立刻停止，避免旧 AI 或旧回放继续改棋盘。

这是一个“版本号式计数”，不是展示给玩家看的步数。

## 3. 本回合步数与行动事务

实现位置：`scripts/GameManager.gd`

核心变量：

```gdscript
var _turn_rollback_stack: Array[Dictionary] = []
var _turn_pending_entries: Array[Dictionary] = []
var _turn_move_count := 0
var _turn_move_kind := ""
```

### 本回合步数 `_turn_move_count`

在 `_apply_local_action(action)` 成功执行本地行动后递增：

```gdscript
_turn_move_count += 1
```

用途：

- `_on_end_turn_requested()` 要求 `_turn_move_count > 0` 才能结束本地玩家回合。
- `_on_piece_clicked()` 中，如果 `_turn_move_count > 0` 且没有强制连跳棋子，则不能再选别的棋子。
- 连跳时会把 `_turn_move_count` 传给 `PassiveSkillRuleEngine.has_jump_action_from()`，用于限制冲刺跳最多两次。
- UI 通过 `GameUI.set_turn_step_count(_turn_move_count, can_undo)` 显示“本回合步数：N”。

### 行动类型 `_turn_move_kind`

第一次行动后记录：

```gdscript
_turn_move_kind = "jump" if was_jump else "step"
```

限制逻辑：

- 如果本回合已经走了普通步，则不能继续行动。
- 如果本回合已经跳跃，则后续只能继续跳，不能改走普通步。

### 回合待提交记录 `_turn_pending_entries`

本地玩家每次行动先进入 `_turn_pending_entries`，直到回合真正结束才提交到全局对局记录：

```gdscript
_turn_pending_entries.append(_build_pending_action_entry(...))
```

提交时 `_commit_turn_pending_entries()` 会：

- 为每条记录补 `index = _get_record_entries().size() + 1`。
- 只把最后一条标记为 `turn_ended = true`。
- 如果本回合获胜，只给最后一条写入 `outcome = "win"`。

### 悔棋栈 `_turn_rollback_stack`

每次本地行动前调用 `_push_turn_rollback_state()` 保存快照。

撤回时：

```gdscript
var snapshot: Dictionary = _turn_rollback_stack.pop_back()
restore_state_snapshot(snapshot)
```

技术效果：

- 只能撤回本回合内尚未提交的行动。
- 一旦 `_clear_turn_transaction()` 被调用，悔棋栈和本回合计数都会清空。

## 4. 棋盘与棋子数量

实现位置：`scripts/BoardManager.gd`

核心配置：

```gdscript
@export var center_radius: int = 4
@export var arm_size: int = 4
```

### 棋盘格数量

`_generate_board_coords()` 分两部分生成棋盘：

- 中心六边形：半径 4，对应 `3r(r+1)+1 = 3*4*5+1 = 61` 格。
- 六个三角臂：每个臂 `4+3+2+1 = 10` 格，六个臂共 60 格。

当前总格数：

```text
61 + 60 = 121
```

`cells` 字典以 `Vector2i` 坐标作为 key 保存所有格子实例。

### 玩家起始区与棋子数量

`_build_player_regions()` 当前只配置两名玩家：

```gdscript
var player_one_start := _get_arm_region(2, 1)
var player_two_start := _get_arm_region(2, -1)
```

每个起始臂 10 个坐标，所以 `reset_pieces()` 会为每个玩家生成 10 枚棋子：

```gdscript
for index in range(coords.size()):
	spawn_piece(player_id, coords[index], "P%d_%02d" % [player_id, index + 1])
```

棋子运行时数量由 `pieces` 字典维护，key 是坐标，value 是棋子节点。

相关统计函数：

- `get_player_pieces(player_id)`：返回某玩家所有棋子。
- `get_player_piece_count(player_id)`：返回某玩家棋子数量。
- `get_pieces_snapshot()`：按坐标排序后导出当前所有棋子，用于存档/回放/回滚。

## 5. 胜负判定计数

实现位置：`scripts/GameManager.gd`

胜利判断函数：

```gdscript
func _did_current_player_win() -> bool:
	var target_region: Array = board_manager.get_target_region(current_player)
	if target_region.is_empty():
		return false

	for coord in target_region:
		var piece = board_manager.get_piece_at(coord)
		if piece == null or piece.player_id != current_player:
			return false
	return true
```

技术含义：

- 不维护分数计数器。
- 直接遍历目标区域的 10 个目标格。
- 只要有一个目标格为空，或不是当前玩家棋子，就不算胜利。

## 6. 被动技能与可达性计数

### 技能分配数量

实现位置：`scripts/BoardManager.gd`

`assign_random_passive_skills(seed_value)` 对每名玩家执行：

- 如果玩家棋子数少于 3，则不分配。
- 按 `piece_id` 排序后使用种子洗牌。
- 取洗牌后的前三枚棋子分配技能：
  - 第 1 枚：`immobilize_aura`
  - 第 2 枚：`dash_jump`
  - 第 3 枚：`freeze_immune`

所以当前默认是每名玩家 3 个特殊棋子，双方合计 6 个特殊棋子。

### 冻结/解冻数量

实现位置：`scripts/BoardManager.gd`

`refresh_piece_skill_status(skill_rules)` 每次刷新所有棋子状态，并统计本次变化：

```gdscript
var frozen_count := 0
var thawed_count := 0
```

返回：

```gdscript
{
	"frozen": frozen_count,
	"thawed": thawed_count,
}
```

用途：

- `AudioManager.play_action()` 根据 `frozen > 0` 播放冻结音效。
- 根据 `thawed > 0` 或移动棋子为破冰棋子播放解冻音效。

### 光环覆盖数量

实现位置：`scripts/PassiveSkillRuleEngine.gd`、`scripts/BoardManager.gd`

光环棋子最多影响周围六格：

```gdscript
for direction in MOVE_DIRECTIONS:
	var coord := center + direction
	if board_manager.has_cell(coord):
		coords.append(coord)
```

`BoardManager.get_all_aura_coverage()` 用字典去重：

```gdscript
var unique_coords: Dictionary = {}
```

所以多个光环覆盖同一格时，观察模式只算一个覆盖格。

### 冲刺跳次数上限

实现位置：`scripts/PassiveSkillRuleEngine.gd`

常量：

```gdscript
const DASH_JUMP_MAX_HOPS := 2
```

限制函数：

```gdscript
func _can_jump(piece, jump_count: int) -> bool:
	return not (
		skills_enabled
		and _get_piece_skill(piece) == SKILL_DASH_JUMP
		and jump_count >= DASH_JUMP_MAX_HOPS
	)
```

效果：

- 普通棋子不受这个上限影响。
- `dash_jump` 棋子本回合跳跃计数达到 2 后，不再生成跳跃动作。
- 这个 `jump_count` 来自本回合动作数或递归路径深度。

### 可达格数量

实现位置：`scripts/PassiveSkillRuleEngine.gd`

`get_turn_reachable_coords(piece, state)` 使用字典作为 set：

```gdscript
var reachable: Dictionary = {}
```

递归收集连跳结果时，每个最终坐标只记录一次。观察模式显示：

```gdscript
"可达 %d 格" % reachable_coords.size()
```

这就是眼睛/观察模式里的可达性计数来源。

## 7. 移动规则的方向计数

实现位置：`scripts/MoveValidator.gd`、`scripts/PassiveSkillRuleEngine.gd`

六边形棋盘使用 6 个轴向方向：

```gdscript
const DIRECTIONS: Array[Vector2i] = [
	Vector2i(1, 0),
	Vector2i(1, -1),
	Vector2i(0, -1),
	Vector2i(-1, 0),
	Vector2i(-1, 1),
	Vector2i(0, 1),
]
```

普通移动：

- 遍历 6 个方向。
- 目标格存在且未占用，则加入可走列表。

普通跳：

- 遍历 6 个方向。
- 中间格有棋子，落点存在且为空，则加入可跳列表。

冲刺长跳：

- 遍历 6 个方向。
- 前方连续两个中间格都有棋子。
- 第三格存在且为空，则生成 `dash_long_jump` 行动。

## 8. 对局记录与回放计数

实现位置：`scripts/GameManager.gd`

### 对局记录结构

`_begin_match_record(layout_name)` 初始化：

```gdscript
{
	"version": 2,
	"players": player_count,
	"random_seed": _match_seed,
	"skills_enabled": _skills_enabled,
	"initial_pieces": board_manager.get_pieces_snapshot(),
	"entries": [],
}
```

### 记录索引

每条行动记录写入：

```gdscript
entry["index"] = _get_record_entries().size() + 1
```

因此记录 index 从 1 开始递增。

UI 显示：

```gdscript
"已记录 %d 步" % _get_record_entries().size()
```

注意：

- 这里的“步”是记录条数，不等同于 `_turn_move_count`。
- 本地玩家一回合多跳时，提交后会形成多条 move 记录。
- 在线模式可能额外记录 `end_turn` 类型，用于标记强制连跳后的主动结束回合。

### 回放步数

变量：

```gdscript
var _replay_step_index := 0
var _replay_play_token := 0
```

`_apply_replay_step(step_index)` 会夹紧范围：

```gdscript
_replay_step_index = clampi(step_index, 0, entries.size())
```

然后从初始棋子状态开始，重新应用前 `_replay_step_index` 条 move 记录。

UI 使用：

```gdscript
game_ui.set_replay_state(_replay_active, _replay_playing, _replay_step_index, _get_record_entries().size())
```

播放时 `_play_replay()` 每隔 `replay_step_delay=0.45` 秒推进一步。

## 9. AI 搜索与评分计数

实现位置：`scripts/AIPlayer.gd`

核心参数：

```gdscript
@export var max_jump_depth: int = 8
@export var jump_bonus: float = 1.7
@export var target_region_bonus: float = 6.0
@export var back_piece_priority: float = 0.22
@export var randomness: float = 0.15
```

AI 选择行动流程：

1. 遍历当前玩家所有棋子。
2. 为每枚棋子收集普通步和跳跃路径。
3. 跳跃路径通过 `_collect_jump_paths()` 递归收集。
4. 当 `depth >= max_jump_depth` 时停止递归。
5. 对每条路径计算分数，取最高分。

跳跃路径加分：

```gdscript
score += max(0, path.size() - 1) * jump_bonus
```

也就是说：

- 单步跳路径 `path.size() == 1`，不吃连跳奖励。
- 每多一段连跳，多加一次 `jump_bonus`。

## 11. 光影、渲染与场景实例数量

实现位置：`scripts/BackgroundManager.gd`

### 帧率与渲染精度

项目配置：

```ini
run/max_fps=60
```

运行时：

```gdscript
@export var target_fps := 60
Engine.max_fps = target_fps
```

渲染精度通过 `render_scale` 控制，当前范围 `0.6 - 1.4`。数值越高越清晰，但 GPU 压力越大。

### 光影预设数量

`LIGHTING_PRESET_ORDER` 当前对 UI 暴露 8 套预设：

1. `performance_clean`
2. `quiet_forest`
3. `balanced_clear`
4. `polished_stone`
5. `soft_day`
6. `overcast`
7. `studio_soft`
8. `supersample_mirror`

另有 `custom` 作为自定义项。

### 棋盘补光

固定 4 个棋盘区域补光：

```gdscript
var placements := [
	Vector3(-6.0, 5.0, -6.0),
	Vector3(6.0, 5.0, -6.0),
	Vector3(6.0, 5.0, 6.0),
	Vector3(-6.0, 5.0, 6.0),
]
```

强度由 `fill_energy * board_fill_scale` 共同决定。

### 棋盘光点

`_build_board_glow_spots()` 当前创建 19 个棋盘光点。

每个光点包含：

- 1 个 `Node3D` 根节点。
- 1 个扁平圆柱 `GlowDot`。
- 1 个弱 `OmniLight3D`。

总计：

```text
19 个可见光斑 + 19 个弱点光源
```

强度由 `board_glow_energy` 控制，每个点还带有自己的 `energy_scale`。

### 石柱/视角锚点发光

实现位置：`scripts/ViewFocusMarkerManager.gd`、`scripts/ViewFocusMarker.gd`

`ViewFocusMarkerManager._build_markers()` 固定创建 6 个视角锚点/石柱：

```gdscript
for index in range(6):
```

每个石柱内部有一个 `MarkerGlowLight`。发光强度由 `marker_glow_energy` 控制。

### 立体草

变量：

```gdscript
@export var grass_blade_count := 720
```

`_build_grass_blade_layer()` 使用 `MultiMesh` 一次性创建 720 个草簇实例。

这是性能优化后的批量实例化，不是 720 个独立节点。

### 大树外圈

`_build_edge_tree_ring()` 创建两层树：

- 外层 24 棵。
- 内层 16 棵。

合计 40 棵高树。

### 森林边缘光

`_build_forest_rim_lights()` 固定创建 6 个森林边缘点光源。

这些光不投影，只负责制造边缘氛围，强度由 `fill_energy * forest_rim_scale * energy_scale` 决定。

### 萤火光点

`_build_firefly_layer()` 固定创建 34 个萤火光点。

这些是发光小网格，不是 34 个真实灯光，所以性能成本远低于点光源。

发光强度由 `firefly_energy` 控制。

### 场景小物件

主要由几个固定数组/循环驱动：

- 基础外围摆件：`_build_scene_props()` 中固定 12 个 placements。
- 棋盘周边花园环：6 个角，每角 4 个小物件，共 24 个。
- 花园环灯笼：6 个。
- 草地细节环：28 个植物/花/草小物件。
- 水流点位：5 个 stream points，每个点两侧各生成石头，共 10 个水边石头。
- 石桥：5 块桥石。
- 树圈：40 棵树。

这些数量都是代码内固定，当前没有暴露成 UI 滑杆。

## 13. 音频计数

实现位置：`scripts/AudioManager.gd`

音效播放器池：

```gdscript
for index in range(6):
	_effect_players.append(_create_player("EffectPlayer%d" % (index + 1), effect_volume_db))
```

播放音效时使用循环索引：

```gdscript
var player := _effect_players[_next_effect_player]
_next_effect_player = (_next_effect_player + 1) % _effect_players.size()
```

技术含义：

- 同时最多复用 6 个音效播放器。
- 后来的音效会覆盖轮到的播放器。
- `_last_effect_time_ms` 按类别做冷却，避免移动、冻结、解冻音效在很短时间内连续爆发。

当前冷却：

- 移动类：85 ms。
- 冻结/解冻类：140 ms。

## 14. 网络包计数

实现位置：`scripts/NetworkManager.gd`

每帧 `_process()` 调用：

```gdscript
_poll_signaling()
_poll_webrtc()
```

信令包读取：

```gdscript
while _websocket.get_available_packet_count() > 0:
```

WebRTC 数据通道读取：

```gdscript
while _channel.get_available_packet_count() > 0:
```

技术含义：

- 每帧把当前已经到达的包全部读完。
- 没有额外维护“总包数”统计。
- ICE candidate 使用 message 内的 `index` 字段原样传递给 WebRTC。

## 15. 需要特别区分的几个“步数”

项目里有三个很容易混淆的“步数”：

| 名称 | 变量/来源 | 重置时机 | 用途 |
| --- | --- | --- | --- |
| 本回合步数 | `_turn_move_count` | 回合结束、重置、回放开始、恢复事务 | 限制本回合行动和冲刺跳次数，UI 显示本回合步数。 |
| 对局记录步数 | `_match_record.entries.size()` | 新对局开始时清空 | 存档、最近记录、回放总步数。 |
| 回放当前位置 | `_replay_step_index` | 开始回放时置 0 | 表示当前回放应用了前几条记录。 |

后续如果要改“最多跳几次”“每回合能不能多走”“回放显示几步”，应先确认要改的是哪一个计数。

## 16. 后续扩展建议

- 如果要支持更多玩家，优先扩展 `BoardManager` 的玩家起始区/目标区，再改 `player_count`。
- 如果要给普通棋子也加跳跃次数上限，应把限制从 `dash_jump` 专属逻辑抽到通用回合规则。
- 如果要继续优化性能，优先把以下数量做成配置项：`grass_blade_count`、树圈数量、萤火数量、棋盘光点数量、森林边缘光数量。
- 如果要做调试面板，可以直接显示 `_turn_move_count`、`_turn_move_kind`、`_turn_rollback_stack.size()`、`_match_record.entries.size()`、`_replay_step_index`。
